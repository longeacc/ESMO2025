Methodology & Implementation Details

  • 1. Ambiguity Detection (High Risk Source): detects values consisting of generic polysemic terms (e.g., "Positive", "Negative", "Detected", "++", "Score 3") or purely numeric values ("10%", "3") that lack context.
  • 2. Self-Description Check (Risk Mitigator): Checks if the extracted text contains the entity's name or specific synonyms (e.g., "HER2 Negative" contains explicit context "HER2"). Explicit values override ambiguity -> Low Risk.
  • 3. Structured Pattern Recognition: Identifies complex formats (e.g., "pT2N0M1") that are inherently specific and unlikely to be ambiguous.
  • Formula: Risk Score = (High Risk Mentions) / (Total Mentions). A score of 0.0 means all extracted values were self-described or unambiguous.
